# v0.2.0 Feature History

## fswatch_auto_ingest

**목적:** `raw/` 파일 변경을 감지하여 10분 주기로 `/ingest`를 자동 실행. 수동 실행 의존성 제거.

**핵심 설계 결정:**
- `fswatch`로 변경 경로를 루트별 queue 파일에 누적 → 타이머 루프(`sleep 600`)에서 일괄 처리
- ingest는 `&` 백그라운드 서브셸로 실행 — 감시 루프와 완전 분리
- lock 파일로 동시 실행 방지, `mv queue queue.tmp` 원자적 drain으로 레이스 컨디션 방지
- launchd PATH 제한 문제 해결: 등록 시점에 실제 바이너리 경로 수집 후 plist에 주입
- 에이전트 선택: `WIKICURATE_AGENT` 환경변수, 기본값 `claude`, Gemini fallback

**리뷰에서 발견된 이슈:**
- `set -u` 환경에서 빈 배열 `seen[*]` unbound variable 오류 → `[[ ${#seen[@]} -gt 0 ]]` 선행 검사로 해결 (v2)

**변경 파일:** `scripts/watch-ingest.sh`(신규), `scripts/watcher.sh`(신규), `deploy.sh`(수정)
