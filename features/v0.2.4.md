# v0.2.4 Feature History

## codex_auto_ingest_support

**목적:** 자동 ingest watcher의 기본 에이전트를 `codex`로 변경하고, Codex 비대화형 실행 흐름을 지원.

**핵심 설계 결정:**
- Codex는 slash command를 네이티브로 실행하지 않으므로 `build_agent_prompt()` 헬퍼로 playbook 기반 자연어 프롬프트 분기 (`_system/commands/ingest.md`를 읽고 실행하라는 형태)
- 에이전트 우선순위: `codex` → `claude` → `gemini` (기존 fallback 유지)
- `watcher.sh register` PATH 수집 대상에 `codex` 추가
- `wiki-schema.md`를 Claude 특정 표현에서 에이전트 중립 표현(WikiCurate Agent Guide)으로 개정

**리뷰에서 발견된 이슈:**
- v2: Codex 옵션 순서 수정, playbook 프롬프트의 shell command substitution 제거, 격리/에러 저장 로직 보강
- v3: sqlite3 `-cmd` 방식 도입 (`busy_timeout` 출력이 retry_count 조회값에 섞이는 문제 수정)
- v4: `.timeout` 방식으로 재수정 (`PRAGMA busy_timeout` 출력도 값 오염 유발)
