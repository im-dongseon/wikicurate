# v0.2.1 Feature History

## auto_lint_after_ingest

**목적:** ingest 배치 완료 후 `/lint`를 자동 실행하여 고아 페이지·끊긴 링크·모순 클레임을 즉시 감지·수정.

**핵심 설계 결정:**
- 트리거 조건: 배치 내 ingest 성공 건수 > 0일 때만 실행 (전부 실패 시 wiki 변경 없으므로 불필요)
- 기존 ingest 서브셸 내부에서 lint 실행 → 동일 lock 공유로 경쟁 조건 없음
- lint 중 다음 주기 도래 시 "연기" 처리 — 의도된 동작

**변경 파일:** `scripts/watch-ingest.sh`(수정)

## version_file

**목적:** `_system/VERSION`을 단일 버전 소스로 도입. 배포 후 볼트에서 버전 확인 가능.

**핵심 설계 결정:**
- 루트 `VERSION` 대신 `_system/VERSION`에 배치 → rsync가 자동 배포, 별도 cp 단계 불필요
- `deploy.sh`가 파일 없으면 오류 종료 — 버전 누락 방지

**변경 파일:** `_system/VERSION`(신규), `deploy.sh`(수정)
