## 배경 및 목적
현재 `watch-ingest.sh`는 `ingest` 실패 시 해당 파일을 포기하여 자동화의 연속성이 끊깁니다. 이를 해결하기 위해 리눅스/macOS 공용 도구인 SQLite를 사용하여 상태를 관리하고, 최종 실패 시 에러 폴더로 격리하는 견고한 재시도 메커니즘을 도입합니다.

## 현행 진단 (결함 목록)
1. **휘발성 처리:** 실패한 파일 경로가 메모리(스크립트 변수)에만 머물다 루프 종료 시 소멸됨.
2. **수동 개입 필수:** 실패한 파일을 다시 처리하려면 사용자가 직접 파일을 건드려야 함.
3. **오류 가시성 부족:** 로그를 보지 않으면 어떤 파일이 왜 처리가 안 되고 있는지 알기 어려움.
