# v0.2.2 Feature History

## ingest_retry_logic

**목적:** ingest 실패 파일을 SQLite DB에 기록하여 다음 주기 자동 재시도(최대 5회). 5회 모두 실패 시 `raw/error/`로 격리.

**핵심 설계 결정:**
- DB 위치: `/tmp` 대신 `$root/_state/data/wikicurate-retries.db` — 재부팅 후에도 재시도 상태 보존
- WAL 모드 + Busy Timeout 5초 + UPSERT: 다수의 서브셸 동시 쓰기 시 데이터 정합성 보장
- DB 조회를 메인 루프에서 수행 — 서브셸 간 동시 조회 충돌 방지, 같은 파일 중복 배분 방지
- 유령 레코드 청소: 메인 루프에서 파일 존재 여부 확인 → 수동 삭제 파일이 DB에 잔류하지 않음
- 격리 시 확장자 보존: `note.md` → `note.20260417_153000.md` (확장자 앞에 타임스탬프 삽입)
- 로그 접두사: `[RETRY N/5]`, `[FAIL]`, `[ISOLATED]`
- `watcher.sh register` 시 `sqlite3` PATH 주입, `watch-ingest.sh` 시작 시도 sqlite3 재검사

**변경 파일:** `scripts/watch-ingest.sh`(수정), `scripts/watcher.sh`(수정)
