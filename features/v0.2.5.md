# v0.2.5 Feature History

## gsheet_auth_simplify

**목적:** GSHEET 인증 3단계 fallback(서비스 계정 → API 키 → URL)을 2단계(ADC → URL)로 단순화.

**핵심 설계 결정:**
- `gcloud auth application-default login` 한 번으로 비공개 시트 접근 가능 → 서비스 계정 JSON, `GOOGLE_API_KEY` 환경변수 불필요
- 스코프를 `spreadsheets.readonly` → `drive.readonly`로 통합 (Sheets/Docs/Slides 공통 적용)

**변경 파일:** `_system/wiki-schema.md`(GSHEET 인증 수정)

---

## gdoc_gslides_ingest

**목적:** `.gdoc`(Google Docs 스텁), `.gslides`(Google Slides 스텁) ingest 정책 신규 도입.

**핵심 설계 결정:**
- 토큰 절약 추출: GDOC — 제목 + 헤딩(H1~H3) + 각 섹션 첫 문단 200자. GSLIDES — 제목 + 슬라이드 제목 + 본문 500자/슬라이드 상한(`SLIDE_BODY_LIMIT`)
- ADC → URL fallback 2단계 구조 (GSHEET와 동일 패턴)
- `drive.readonly` 단일 스코프로 Sheets/Docs/Slides 통합

**리뷰에서 발견된 이슈:**
- GSLIDES 본문 무제한 추출 위험 → `SLIDE_BODY_LIMIT = 500` 상한 추가
- 선행 feature(`gsheet_auth_simplify`) 완료 후 이 feature의 GSHEET 스코프 추가 수정은 additive 변경으로 충돌 없음 확인

**변경 파일:** `_system/wiki-schema.md`(GDOC·GSLIDES 섹션 추가, GSHEET 스코프 수정, log.md 규칙 추가), `_system/commands/setup.md`(Section 3 Google 연동 신설, 번호 재정렬)

## docx_ingest

**목적:** `.docx`(Microsoft Word) ingest 정책 신규 도입. 인증 불필요 — 로컬 파일 직접 파싱.

**핵심 설계 결정:**
- 토큰 절약 추출: 제목 + 헤딩 구조 + 각 섹션 첫 단락 200자 + 표 개수(`len(doc.tables)`)
- 스타일 매칭: 영문("Heading 1") 및 한국어("제목 1") 모두 대응 (`startswith` 방식). 커스텀 스타일은 지원 범위 밖 → 표준 스타일 권장 문구 추가
- 표 내용은 추출하지 않고 개수만 기록 — XLSX의 헤더+행수 패턴과 동일한 접근

**변경 파일:** `_system/wiki-schema.md`(DOCX 섹션 추가, log.md 규칙 추가), `_system/commands/setup.md`(python-docx 추가)

## gdrive_sa_auth (중간 단계 — OAuth로 대체됨)

**목적:** ADC(gcloud) 차단 이후 SA(서비스 계정) 방식으로 교체 시도.

**결과:** SA 방식도 Google Drive API 접근 제한 문제로 실사용 불가 확인 → 동일 릴리즈 내에서 OAuth 방식(`google_oauth_profile`)으로 대체됨. SA 관련 코드는 현재 `wiki-schema.md`에 존재하지 않음.

**추후 참고:** SA는 공유 드라이브(Shared Drive) 파일에 대한 접근이 제한되는 경우가 있어 개인 OAuth 키 방식이 적합했음.

## google_oauth_profile

**목적:** 위키별로 다른 Google 계정을 사용할 수 있도록 OAuth 프로필 기반 인증 도입.

**핵심 설계 결정:**
- 위키 루트의 `.wikicurate` 파일에 `google_profile` 지정 → `~/.config/wikicurate/client_secret_{profile}.json` + `token_{profile}.pickle` 선택
- `_find_wiki_root()`: CWD에서 상위로 탐색해 `wiki/index.md`가 있는 디렉토리 반환 (하위 디렉토리 실행 안전)
- `_get_google_creds()`: `.wikicurate` 없음 / profile 미설정 / 인증 실패 → `None` 반환 (예외 없음) → 호출부 `except` 블록에서 fallback 처리
- GSHEET/GDOC/GSLIDES 세 섹션이 동일 헬퍼 호출 — 인증 로직 중복 없음

**현재 프로필 설정 상태:**
- `Google Drive/wiki-agent/` — `.wikicurate`: `{"google_profile": "work"}`
- `Google Drive/SynapseModule/Docs/` — `.wikicurate`: `{"google_profile": "work"}`
- `Documents/wikicurate/` — `.wikicurate` 없음 → fallback (개인 credentials 미발급)

**추후 개발:**
- 개인 계정 credentials 발급 시: `~/.config/wikicurate/client_secret_personal.json` 추가 + `Documents/wikicurate/.wikicurate` 생성(`{"google_profile": "personal"}`)
- 신규 위키 추가 시: 위키 루트에 `.wikicurate` 생성하고 해당 프로필 credentials 파일 준비

**변경 파일:** `_system/wiki-schema.md`(Google 인증 공통 로직 블록 신설, GSHEET/GDOC/GSLIDES 1단계 교체)
