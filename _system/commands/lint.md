wiki 전체를 대상으로 건강 점검 및 자동 수정을 수행한다.

1. **폴더 구조 준수 확인**: wiki 페이지가 정의된 카테고리 폴더에만 있는지 검사한다.
   - 허용 폴더: `wiki/sources/`, `wiki/entities/`, `wiki/concepts/`, `wiki/analyses/`
   - `wiki/` 루트에 직접 저장된 페이지(index.md, log.md 제외)는 적절한 폴더로 이동한다.
   - 정의되지 않은 폴더에 있는 페이지는 사용자에게 보고 후 이동 여부를 결정한다.

2. **그래프 기반 구조적 문제 감지**: `graphify-out/graph.json`이 있으면 그래프로 탐지한다.
   - 인바운드 엣지가 없는 노드 → 고아 페이지
   - 존재하지 않는 노드를 가리키는 엣지 → 끊긴 링크
   graph.json이 없으면 `wiki/index.md`와 전체 페이지 순회로 폴백한다.

3. **내용 문제 점검**: 각 페이지를 읽으며 확인한다.
   - 페이지 간 모순되는 클레임
   - 더 최신 소스로 인해 무효화된 내용
   - 본문에 언급되지만 페이지가 없는 개념·엔티티
   - 웹 검색으로 보완 가능한 데이터 공백

4. **자동 수정**:
   - 고아 페이지: 관련 페이지에 교차 참조 추가
   - 끊긴 링크: 링크 제거 또는 대상 페이지 생성
   - 언급된 개념에 페이지 없음: 스텁 페이지 생성 후 index.md 갱신
   - 모순·무효화된 클레임: 해당 페이지에 충돌 내용 명시 후 최신 정보로 갱신

5. `wiki/log.md` 끝에 추가한다: `## [오늘날짜] lint | 수정된 항목 수 요약`
